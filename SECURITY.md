# Security Policy

## Supported Versions

Security fixes are applied to the latest published minor release.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability or leaked publishing credential. Use GitHub's private vulnerability reporting feature for this repository. Include the affected version, reproduction steps, impact, and any suggested mitigation.

PyPI releases use GitHub Actions Trusted Publishing. Long-lived PyPI tokens must not be committed, attached to issues, or stored in workflow files.
