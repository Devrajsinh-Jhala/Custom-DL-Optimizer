Custom-DL-Optimizer

A regression-aware PyTorch inference optimizer that profiles candidate plans, rewrites supported graphs, validates numerical output, and keeps the fastest safe path.

Compiler pipeline

Optimization with an exit strategy.

Every candidate must remain numerically close to eager FP32. Custom graph work is retained only when pilot timing clears the configured gain threshold.

01

Profile

Capture representative operator timing and the input signature.

02

Rewrite

Fold safe Conv-BN patterns and replace supported FX nodes.

03

Validate

Reject candidates that fail output structure or parity checks.

04

Select

Keep FX or Inductor only when measured speedup beats native.

One-call interface

Inspect the decision, not just the output.

The original `optimize()` workflow remains. `optimize_with_report()` adds the selected plan, timing, graph rewrite counts, numerical error, and fallback reason.

Read the usage guide
quickstart.py 
from custom_dl_optimizer import (
    AutoOptimizer,
    OptimizationConfig,
)

config = OptimizationConfig(
    enable_compile=True,
    compile_mode="max-autotune",
    min_speedup=1.02,
)

optimizer = AutoOptimizer(model, config=config)
optimized, report = optimizer.optimize_with_report(inputs)

print(report.selected_plan)
print(report.selection_reason)
